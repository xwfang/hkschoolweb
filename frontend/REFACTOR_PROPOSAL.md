# 前端（家长端）页面功能优化方案

基于目前的实现和 API 能力，建议对家长端的功能结构进行以下优化，使其更符合“**择校 -> 申请 -> 追踪**”的核心用户旅程，并更自然地融入 AI 能力。

## 1. 核心导航结构 (Bottom Navigation)

建议将底部导航调整为以下四个核心模块：

| Tab 名称 | 对应页面 | 核心定位 | 关键变化 |
| :--- | :--- | :--- | :--- |
| **择校 (Discover)** | `HomePage` | 发现与筛选学校 | **新增顶部子女切换器**；整合“智能推荐”与“高级筛选”。 |
| **AI 助手 (Assistant)** | `ChatPage` | 咨询与辅助 | 增加**场景化快捷指令**（如：生成自荐信、模拟面试）。 |
| **进度 (Tracking)** | `TrackingPage` | 申请进度管理 | 增加**时间轴/截止日期提醒**；优化状态展示。 |
| **我的 (Me)** | `ProfilePage` | 档案与设置 | 侧重于**档案管理**和账号设置；弱化日常切换功能。 |

---

## 2. 页面详细规划

### 2.1 Tab 1: 择校 (Discover) - *首页重构*
**痛点解决**: 之前的首页搜索功能较弱，且切换子女需要在“个人中心”操作，路径太长。

*   **顶部栏 (Header)**:
    *   **子女切换器**: 下拉菜单直接切换当前关注的子女（例如：`正在为 [Jason] 择校`）。
    *   **状态概览**: 显示该子女的“已关注/已报名”数量小红点。
*   **核心区域**:
    *   **AI 匹配卡片**: 展示 Top 1 最匹配的学校，并附带 AI 推荐理由（“匹配度 95%：该校重视理科，与 Jason 的数学特长吻合”）。
    *   **快捷筛选区 (Chips)**: `Band 1`, `男校`, `九龙城`, `直资` 等标签，点击即搜。
    *   **学校列表流**: 混合展示推荐学校和热门学校。
*   **交互优化**:
    *   搜索框支持**自然语言搜索**（例如：“学费便宜的直资小学” -> 调用 AI 解析 -> 搜索）。

### 2.2 Tab 2: AI 助手 (Assistant)
**痛点解决**: 纯对话界面对不知道问什么的用户不友好。

*   **默认界面**:
    *   **欢迎语**: "我是您的升学顾问，针对 [Jason] 的情况，您可以问我..."
    *   **快捷指令 (Quick Actions)**:
        *   📝 **写自荐信**: 根据档案自动生成。
        *   🎤 **面试模拟**: 针对已关注学校进行模拟提问。
        *   📊 **简历分析**: 分析当前档案的优劣势。
*   **对话界面**: 保持现有的聊天体验，但支持**引用学校**（例如在学校详情页点击“问问 AI”，带入学校上下文）。

### 2.3 Tab 3: 进度 (Tracking)
**痛点解决**: 只有列表，缺乏时间紧迫感和下一步指引。

*   **视图优化**:
    *   **状态分组**: 将申请按状态分组（`关注中` -> `准备中` -> `已递交` -> `面试/结果`）。
    *   **关键日期**: 在卡片上高亮显示“截止日期”或“面试时间”。
*   **功能增强**:
    *   **下一步提示**: 针对每个状态，提供 AI 建议（例如：状态为“面试中” -> 提示“查看去年面试真题”）。

### 2.4 学校详情页 (School Detail) - *二级页面*
*   **增强**:
    *   **匹配度分析**: 增加一个 AI 分析模块，详细解释该校与当前子女的匹配点和风险点。
    *   **一键填表**: 如果学校有在线表格（未来扩展），支持一键将子女档案填入。
